## Алгоритмы
класс Solution (algorythms.py)
- `two_pointers`: O(M + N) время, O(1) память.
- `binary_search`: O(M log N) время, O(1) память.
- `exponential_search`: O(M log N) время (в среднем), O(1) память.
- `binary_divide`: O(M log(2N/M)) время, O(log M) память.

## Данные:
### /data/
500 датасетов разной длины
(40% — до 10k, 50% — до 100k, 10% — до 1M элементов).

## Графики:
### /charts
номер соответствует номеру датасета, на котором прогоняли алгоритмы

## Структура проекта
- **`data/`**: Сгенерированные датасеты. Каждый файл — `test_num.npy` (NumPy-архив с кортежем `(arr_m, arr_n)`). Датасеты отсортированы, уникальны; чередование: четные — без общих элементов, нечетные — с одним общим.
- **`charts/`**: HTML-файлы с графиками сравнения (время и память для каждого теста). Файл: `test_num.html` (Plotly).
- **`reports/`**: JSON-отчеты по тестам. Файл: `test_num.json` с результатами методов (результат, время, память).

### Скрипты
- **`algorythms.py`**: логика алгоритмов (2 указателся, бинарный поиск, экспоненциальный поиск, бинпоиск с разделением)
- **`generate_data.py`**: генерация случайных отсортированных массивов

- **`tests.py`**: запуск тестов (подразумевается, что данные уже есть)
